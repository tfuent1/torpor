# Phase 2 — Collections & Workspaces

## Goal
Organise requests into collections and workspaces. Enable the git-friendly team sharing workflow.

## Deliverables

- [ ] Workspace file support (load/save workspace.yaml)
- [ ] Collection file support (load/save collection.yaml)
- [ ] Sidebar with workspace tree (collections and requests)
- [ ] Create, rename, delete collections and requests from TUI
- [ ] Request ordering within collections
- [ ] Collection-level auth inheritance
- [ ] Collection-level header inheritance
- [ ] Import Postman collection format
- [ ] Import Insomnia collection format

## Definition of Done

A developer can organise requests into named collections, commit the workspace directory to a git repository, and a teammate can clone it and immediately see all requests in the TUI.
