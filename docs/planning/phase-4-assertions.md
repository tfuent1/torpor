# Phase 4 — Assertions & Test Runner

## Goal
Make Torpor useful for API regression testing.

## Deliverables

- [ ] Assertion definitions parsed from request YAML files
- [ ] Status code assertions
- [ ] Response time assertions
- [ ] Header value assertions
- [ ] JSONPath body assertions
- [ ] Per-request assertion results displayed in TUI
- [ ] Collection runner — execute all requests in a collection sequentially
- [ ] Collection runner pass/fail summary

## Definition of Done

A developer can define expected behaviour in request files and run a collection to verify an API behaves correctly, with a clear pass/fail report per request.
